## рҹҺҜ Features

| Module | Description |
|--------|-------------|
| рҹ”‘ **Hunter.io API** | Queries Hunter.io domain-search API вҖ” returns emails with confidence scores |
| рҹ“¬ **Snov.io API** | Async Snov.io domain-search вҖ” polls until complete, follows pagination |
| рҹ“ң **crt.sh Lookup** | Enumerates subdomains via Certificate Transparency logs |
| рҹ”Қ **Dork Search** | Queries DuckDuckGo with OSINT dorks to find exposed emails |
| рҹҢҗ **Web Crawler** | Concurrently crawls target domain pages to extract email addresses |
| рҹ’ҫ **Multi-format Output** | Save results as `.txt`, `.json`, or `.csv` |
| вҡЎ **Concurrent** | Multi-goroutine crawling for fast results |
| рҹҺЁ **Colorized Output** | Beautiful terminal output with ASCII banner |

## рҹҡҖ Usage

```
email-hunter -d <domain> [options]
```

### Options

| Flag | Description | Default |
|------|-------------|---------|
| `-d <domain>` | **Target domain (required)** | вҖ” |
| `-o <file>` | Output file (`.txt` / `.json` / `.csv`) | вҖ” |
| `-p <int>` | Max pages to crawl | `50` |
| `-depth <int>` | Crawl depth | `3` |
| `-hunter-key <key>` | Hunter.io API key *(overrides config file)* | вҖ” |
| `-snov-key <key>` | Snov.io API key *(overrides config file)* | вҖ” |
| `--no-hunter` | Disable Hunter.io module | вҖ” |
| `--no-snov` | Disable Snov.io module | вҖ” |
| `--no-web` | Disable web crawler module | вҖ” |
| `--no-dork` | Disable dork search module | вҖ” |
| `--no-cert` | Disable crt.sh module | вҖ” |

### Examples

```bash
# First run вҖ” auto-creates config file, fill in keys then re-run
./email-hunter -d example.com

# Full scan using keys from config file
./email-hunter -d example.com

# Save results as JSON
./email-hunter -d example.com -o results.json

# API-only mode (fastest, no crawling needed)
./email-hunter -d example.com --no-web --no-dork --no-cert

# Free modules only вҖ” no API keys required
./email-hunter -d example.com --no-hunter --no-snov

# Override a key without editing the config file
./email-hunter -d example.com -hunter-key MY_OTHER_KEY
```
